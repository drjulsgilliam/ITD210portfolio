GitHub Classroom Guide

ITE 270 — Advanced Web Design

This guide walks you through everything you need to get started with GitHub Classroom for this course. You will learn how to create a free GitHub account, accept assignments, download your files, and upload your completed work — all from the browser. No command line or Git software is required.

Create a Free GitHub Account

If you do not already have a GitHub account, start here:

  1. Go to github.com.
  2. Click "Sign up" in the top-right corner.
  3. Enter your email address, create a password, and choose a username. Use something professional — this may appear on your resume someday (example: jane-doe-dev or jdoe2026).
  4. Complete the verification steps and confirm your email.
Already have an account?

If you already have a GitHub account from a previous class or personal use, you do not need to create a new one. Just make sure you remember your login credentials.

Accept the GitHub Classroom Assignment

GitHub Classroom automatically creates a personal repository for you when you accept an assignment. You do not need to create your own repository — it is done for you.

ITE 270 — Accept Your Assignment

Accept Assignment on GitHub Classroom
1

Click the Assignment Link

  1. Click the assignment link above (or the link posted in Canvas).
  2. If you are not already signed in to GitHub, you will be prompted to sign in.
  3. You may be asked to select your name from a class roster — find and click your name to link your GitHub account to the course.
  4. Click "Accept this assignment".
  5. Wait a few seconds — GitHub will create your personal repository. You will see a confirmation page with a link to your new repo.
2

Open Your Repository

  1. Click the link to your new repository (it will look something like github.com/YOUR-ORG/assignment-name-your-username).
  2. You should see the assignment files already in the repository — this is your personal copy to work with.
  3. Bookmark this page. You will come back to it every time you need to upload or check your work.

Download Your Files

To work on the assignment on your computer, download the files from your repository:

3

Download as ZIP

  1. On your repository page, click the green "<> Code" button near the top of the file list.
  2. Click "Download ZIP".
  3. Find the downloaded .zip file on your computer (usually in your Downloads folder).
    • Windows: Right-click the .zip file and choose "Extract All…". Then move the extracted folder to your Documents or Desktop so you are not working inside the Downloads folder. Important: After extracting, make sure you do not have a folder inside a folder. If you see something like my-project > my-project > index.html, move the inner contents up one level so index.html is directly inside your project folder.
    • Mac: Double-click the .zip file to extract it automatically, then move the extracted folder to your Documents or Desktop.
  4. Open the extracted project folder in Visual Studio Code. Make sure you can see your project files (like index.html) at the top level of the file list — if they are nested inside another folder, you opened the wrong level.
Avoid the folder-in-a-folder problem.

When Windows extracts a ZIP file, it often creates a wrapper folder with the same name. You end up with Desktop > my-project > my-project > index.html instead of Desktop > my-project > index.html. If this happens, open the inner folder, select all the files, and move them up one level. Then delete the empty outer folder. In VS Code, your file tree should show your project files directly — not a single folder containing everything.

Upload Your Completed Work to GitHub

When you are ready to submit your work (or save your progress), upload your files back to GitHub directly through the browser. No Git commands or software needed.

4

Upload Files Through the Browser

  1. Go to your repository page on GitHub (the one that was created when you accepted the assignment).
  2. Click "Add file" in the top-right area of the file list.
  3. Choose "Upload files" from the dropdown menu.
  4. You will see a large drag-and-drop area. You have two options:
    • Drag and drop: Open the project folder on your computer and drag all your files and folders directly into the browser upload area.
    • Browse: Click "choose your files" and select the files you want to upload.
  5. Wait for all files to finish uploading (you will see a progress bar and file names appear).
  6. In the "Commit changes" section at the bottom of the page, type a short description of what you did (example: "Completed assignment" or "Added styling and images").
  7. Click the green "Commit changes" button.
Upload tip:

When uploading updated files, GitHub will automatically replace old versions with your new ones. Make sure you upload all of your project files — not just the ones you changed — so nothing gets out of sync. It is safer to upload everything each time.

Save your work often.

Do not wait until the last minute to upload. Upload your files to GitHub regularly throughout the project. This protects your work in case something happens to your computer, and it lets the professor see your progress.

Verify Your Upload

After uploading, always double-check that everything looks correct:

  1. Go back to your repository's main page on GitHub.
  2. Confirm that you can see all your files listed (HTML, CSS, images, media, etc.).
  3. Click on your index.html (or main file) to verify it contains your latest code — not the original template.
  4. Check that your images folder contains the images you added.

If something looks wrong or files are missing, simply repeat the upload process. Uploading again will overwrite the old versions.

Common Issues and Fixes

I cannot find my repository

Go to github.com, sign in, and look in the left sidebar under "Repositories". Your assignment repo should appear there. You can also search for it using the search bar at the top of the page.

I accidentally uploaded the wrong files

No problem — just upload the correct files again. GitHub will overwrite the previous versions. Your most recent upload is always what the professor sees.

My folder structure looks wrong after extracting

If you see a folder inside a folder (like assignment-name > assignment-name > files), open the inner folder, select everything inside it, cut or copy it, go up one level, and paste. Then delete the now-empty inner folder. When you open the project in VS Code, your files should appear directly in the sidebar — not inside a nested folder.

GitHub says my file is too large

GitHub has a file size limit of approximately 25 MB per file. If you have a video or large image that exceeds this limit, consider compressing it or hosting it externally (such as uploading a video to YouTube as unlisted and embedding it with an <iframe>).

I see a 404 error when clicking the assignment link

Make sure you are signed in to your GitHub account. If you are signed in and still see an error, try opening the link in a different browser or clearing your browser cache.

Quick Reference — Your Workflow

  1. Accept the assignment using the GitHub Classroom link.
  2. Download the ZIP from your repository (green "Code" button → "Download ZIP").
  3. Extract the ZIP and move the folder to Documents or Desktop.
  4. Open the folder in Visual Studio Code and do your work.
  5. Upload your files back to GitHub ("Add file" → "Upload files" → drag and drop → "Commit changes").
  6. Verify your files appear correctly on GitHub.
  7. Submit your GitHub repository URL in Canvas.
